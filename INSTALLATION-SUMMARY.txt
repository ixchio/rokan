INSTALLATION METHODS:
====================

METHOD 1: User Installation (Recommended)
──────────────────────────────────────────
cd rokan-skills
./install-rokan.sh

Creates:
  ✓ Binary: ~/.local/bin/rokan
  ✓ Virtual Environment: ~/.local/opt/rokan/venv
  ✓ Service: ~/.config/systemd/user/rokan.service
  ✓ Desktop Launcher: ~/.local/share/applications/rokan.desktop
  ✓ Data Directory: ~/.rokan/

Usage:
  rokan              # Launch TUI
  rokan ask "..."    # CLI access


METHOD 2: System-Wide Installation
───────────────────────────────────
cd rokan-skills
sudo ./install-rokan.sh system

Creates:
  ✓ Binary: /usr/local/bin/rokan
  ✓ Virtual Environment: /opt/rokan/venv
  ✓ Service: /etc/systemd/system/rokan.service
  ✓ Desktop Launcher: /usr/share/applications/rokan.desktop
  ✓ Data Directory: ~/.rokan/ (still per-user)

Usage:
  Same as user mode but available to all users

SYSTEM SERVICE INTEGRATION:
===========================

Enable Auto-Start (User Mode):
  systemctl --user enable rokan
  systemctl --user start rokan

Start Service:
  systemctl --user start rokan

Stop Service:
  systemctl --user stop rokan

Check Status:
  systemctl --user status rokan

View Logs:
  journalctl --user -u rokan -f

For System Mode, replace "--user" with "sudo"

TROUBLESHOOTING:
================

"rokan: command not found"
  → Add to PATH: export PATH="$HOME/.local/bin:$PATH"

Python version mismatch
  → Ensure Python 3.10+: python3 --version

Installation permission denied
  → Make script executable: chmod +x install-rokan.sh

API key errors
  → Set: export NVIDIA_API_KEY="nvapi-..."

Service won't start
  → Check logs: journalctl --user -u rokan -f

See INSTALL.md for more detailed troubleshooting
